# Meta'gor

A single-pane-of-glass dashboard for **World of Warcraft: Midnight** Mythic+ best-in-slot gear, talents, consumables, and content creators. Gear and stats are **live-aggregated** from Blizzard's connected-realm Mythic+ leaderboards and resolved through the official Blizzard Game Data API. Consumables, rotations, and creator links are **hand-curated**.

The site is fully static — no backend, no server. Data is regenerated by a scheduled Node script on a homelab and committed; Cloudflare Pages (or any static host) serves the result.

## Stack

- **Frontend** — vanilla ESM (no framework), 3-pane layout lifted from the local `public/loktar/` prototype, Wowhead tooltips via `power.js`.
- **Backend** — Node.js 20+ ESM, scheduled on Windows Task Scheduler on a homelab.
- **Data source** — Blizzard Developer Portal (free, client_id + client_secret). **Raider.IO was originally planned but is no longer used** — their free API no longer exposes a top-N-by-spec endpoint, so the scraper now derives top performers from Blizzard's `connected-realm/{id}/mythic-leaderboard/{dungeonId}/period/{periodId}` endpoint.
- **Hosting** — Cloudflare Pages (planned) on `metagor.logansworld.co.za`.

## Provision (one-time)

1. Sign in at https://develop.battle.net (any Battle.net account works).
2. **My Account → API Access → Create a new client**.
3. **Name:** `Meta'gor`. **Service:** World of Warcraft. **Redirect URL:** blank.
4. Submit. Copy the **Client ID** and **Client Secret** (the secret is only shown once).
5. Paste them into `.env` at the project root:
   ```
   BLIZZARD_CLIENT_ID=...
   BLIZZARD_CLIENT_SECRET=...
   ```

## Run the credential sanity check

Confirms Blizzard accepts the credentials. **No scraping, no data written.**

```
node scripts/verify-credentials.mjs
```

A green light = ready to run the scraper.

## Run the static site locally

```
cd public
npx http-server -p 8080
```

Or any other static file server pointed at `public/`.

## Update data

```
node scripts/run-once.mjs --dry-run            # discovery only, no fetches
node scripts/run-once.mjs demon-hunter-havoc   # one spec only
node scripts/run-once.mjs                      # all 39 specs (slow)
```

The orchestrator:
1. Discovers the current M+ season + period + active dungeons from Blizzard.
2. Fetches all 736 active `connected-realm/.../mythic-leaderboard` endpoints.
3. Tallies the top performers per spec across all leaderboards.
4. Fetches Blizzard character equipment + specializations for the top 50 candidates.
5. Aggregates into `data/aggregated_bis.json`.
6. Resolves consumable item names (writes back to `guides.json`).
7. Validates against `data/schema/aggregated_bis.schema.json`.
8. Atomic-writes the result.

After the data is fresh:
```
git add data/aggregated_bis.json
git commit -m "data: refresh aggregated_bis"
git push                          # Cloudflare Pages auto-rebuilds
```

## Known operational constraints (June 2026 / patch 12.0.7)

- **Expansion name and patch number are not exposed** via Blizzard's API anymore. The frontend will render "Unknown" for those fields.
- **The current M+ period is `1068`** (auto-discovered), season is `17`.
- **Active dungeons auto-probe** to 8 per week (one connected realm probes 82 dungeons; ~200 are 200 OK, ~600 are 404; we keep the ones that returned 200).
- **Top-of-leaderboard characters often have private profiles** (Blizzard 403). The orchestrator uses a 200-candidate buffer and a 75-attempt cap to mitigate.
- **The Blizzard profile API throttles after high-volume leaderboard fetches** (observed: 736 leaderboards → profile API 403s for several minutes). The scraper pauses 3s after leaderboards and runs sequentially with 400ms between profile fetches. If you see many 403s, wait 5-10 minutes and re-run.

## Notes for the next agent

- The frontend currently reads `data/aggregated_bis.json` and `data/guides.json` at runtime. Until the scraper produces a real `aggregated_bis.json`, the file may be the schema-valid fixture from a previous run.
- Spec keys use the form `<class-slug>-<spec-slug>`, e.g. `demon-hunter-havoc`. Keep this convention.
- The `public/loktar/` folder is the original prototype and is kept untouched for visual reference.
- Wowhead tooltip behavior is wired through `data-wowhead` attributes on item/spell elements. `app.js` calls `ensureWowheadScript()` to inject `https://wow.zamimg.com/js/tooltips.js?domain=europe&power` exactly once.
