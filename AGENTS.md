# AGENTS.md

Operating notes for AI/agent sessions working in this repo. Read first.

## Hard rules

1. **Never write Blizzard credentials to a file that gets committed.** Only `.env` is allowed to hold them, and it is gitignored.
2. **Never hardcode the expansion, patch, or season ID anywhere.** They must come from `discover.mjs` at scraper boot. If you see a hardcoded `12.0.7`, `midnight`, or `17` in `scripts/`, it's a bug. (The period `1068` is the most recently discovered value; treat it as an example, not a constant.)
3. **Never commit `data/aggregated_bis.json` by hand.** It is generated by `scripts/run-once.mjs`. Edits go through the scraper.
4. **`scripts/verify-credentials.mjs` must remain token-only.** No scraping, no `aggregated_bis.json` write, no spec fetch. If you find yourself wanting to "just fetch one profile" in the verifier, create a new script instead.
5. **Raider.IO is NOT a data source.** The plan originally called for it; their free top-N-by-spec endpoint was removed, so the scraper now uses Blizzard's connected-realm leaderboards. Do not re-add Raider.IO code.

## Daily commands

```
# Sanity check Blizzard OAuth
node scripts/verify-credentials.mjs

# Discover current season/period without fetching anything
node scripts/run-once.mjs --dry-run

# Run for a single spec
node scripts/run-once.mjs demon-hunter-havoc

# Synthetic smoke test (no network)
node scripts/smoke-aggregate.mjs

# Serve the static site
cd public && npx http-server -p 8080
```

## File touch map

- `data/aggregated_bis.json` — auto-generated. Do not hand-edit.
- `data/guides.json` — hand-curated. OK to edit.
- `data/schema/*` — JSON Schema. Update when contract changes (and bump `schema_version`).
- `public/js/registry.js` — class/spec catalog. Single source of truth for class colors, armor types, and slot layout.
- `public/js/app.js` — entry point. Fetches two JSONs, dispatches to renderers.
- `public/js/render/*.js` — one renderer per UI block. Keep them pure: input data, mutate the host.
- `public/styles/*` — visual layer.
- `scripts/lib/config.mjs` — non-volatile config (region, locale, namespaces, rate limits, spec catalog).
- `scripts/lib/cache.mjs` — disk-backed TTL cache.
- `scripts/lib/blizzard.mjs` — Blizzard OAuth + profile + static item.
- `scripts/lib/leaderboard.mjs` — connected-realm leaderboard fetcher, top-performer tally.
- `scripts/lib/discover.mjs` — boot-time version discovery.
- `scripts/lib/aggregate.mjs` — pure tally logic.
- `scripts/lib/resolve-consumables.mjs` — Blizzard static item name lookup.
- `scripts/lib/schema-validate.mjs` — AJV validator.

## Spec ID convention

Always `<class-slug>-<spec-slug>`, e.g. `demon-hunter-havoc`, `mage-arcane`. Used as the key in both `aggregated_bis.json` and `guides.json`. Cross-reference: `scripts/lib/config.mjs` SPECS array has both our slug and Blizzard's `blizzardClassId` / `blizzardSpecId` for each.

## Slot names

The 16 equipment slots: `head neck shoulders back chest wrists hands waist legs feet finger1 finger2 trinket1 trinket2 mainhand offhand`. The renderer maps to display labels; the data file uses these keys.

## Operational note: Blizzard throttling

After 736 leaderboard calls in a single script invocation, Blizzard's profile API starts returning 403 for several minutes. The orchestrator mitigates this by:
- Caching leaderboards to disk for 12h.
- Pausing 3s after the leaderboard batch.
- Fetching profiles sequentially with 400ms between each.

If you see many `403` errors in a run, the throttle is real; wait 5-10 minutes and re-run. The cache means the next run will skip the leaderboard batch entirely.

## Deploy

Cloudflare Pages auto-rebuilds on `git push` to the connected branch. No build step — `public/` is the publish directory.
